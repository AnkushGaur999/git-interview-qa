## üß≠ Local vs Remote Repository

| Repository Type | Description                                                                                                                |
| --------------- | -------------------------------------------------------------------------------------------------------------------------- |
| **Local**       | Your personal copy of the repo on your machine (`.git/` folder inside).                                                    |
| **Remote**      | A shared version of the repository hosted on a platform like **GitHub**, **GitLab**, or **Bitbucket**. Accessible via URL. |

> Git allows you to **sync** your local repo with the remote using commands like `clone`, `pull`, `push`, etc.

---

## üîπ 1. `git clone` ‚Äì **Copy a Remote Repo to Your Local Machine**

```bash
git clone <remote-repo-url>
```

‚úÖ This:

* Downloads the entire repo including history
* Creates a local copy with a remote named `origin`

**Example:**

```bash
git clone https://github.com/user/repo.git
```

---

## üîπ 2. `git remote` ‚Äì **Manage Remote Connections**

### View current remotes:

```bash
git remote -v
```

### Add a new remote:

```bash
git remote add origin <remote-url>
```

### Remove a remote:

```bash
git remote remove origin
```

> `origin` is the default alias for the main remote repo.

---

## üîπ 3. `git fetch` ‚Äì **Download Changes From Remote (No Merge)**

```bash
git fetch origin
```

‚úÖ Downloads updates (branches, commits) from the remote, but **does NOT merge** into your working branch.

> Use it when you want to see what‚Äôs new before merging.

---

## üîπ 4. `git pull` ‚Äì **Fetch + Merge Remote Changes**

```bash
git pull origin main
```

‚úÖ Downloads **and merges** the latest changes from the remote to your local branch.

> Shortcut for: `git fetch` + `git merge`

---

## üîπ 5. `git push` ‚Äì **Upload Local Changes to Remote**

```bash
git push origin main
```

‚úÖ Sends your **commits** from local branch to the **remote branch**.

> First time pushing to a new remote branch:

```bash
git push -u origin main
```

The `-u` sets upstream tracking so future pushes can just use `git push`.
